## 2. 提交建议

1. 将本目录整体放入仓库 `docs/`。
2. 根目录 `README.md` 中保留文档导航链接。
3. 提交前把文档中的 `____`、成员 A/B/C/D/E、仓库地址占位替换为真实信息。
4. 截图不建议直接放入 Markdown 内大段粘贴，可统一存入 `docs/assets/`，并在最终 Word 报告中插入。
5. `ai.md`、`assign.md` 是评分重点之一，应尽量保留真实过程和人工修改痕迹。

## 3. 当前代码状态摘要

- 后端：FastAPI + SQLAlchemy + Pydantic，包含认证、数据、任务、导出、监控、SQL、系统管理、WebSocket 等路由。
- 前端：Vue 3 + Vite + Element Plus + ECharts，包含登录、驾驶舱、股票、基金、公告、任务、数据源、数据质量、监控、元数据、SQL、导出、系统管理、审计等页面。
- 测试：`backend/tests` 共 5 个主要测试文件，定义 65 个 pytest 用例，覆盖清洗、接口、监控、权限、SQL 防护、告警、租户隔离、导出隔离等关键路径。
- 部署：默认 SQLite 克隆即跑，可通过 `.env` 切换 MySQL；真实数据源不可用时自动回退样例数据，保证答辩稳定。
