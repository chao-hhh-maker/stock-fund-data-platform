## 1. 使用前准备

请先按 `install.md` 启动后端与前端。浏览器访问：`http://localhost:5173`。

推荐演示账号：

| 角色 | 用户名 | 密码 | 适合演示 |
| --- | --- | --- | --- |
| 管理员 | `admin` | `admin123` | 全部功能、采集、SQL、系统管理、审计 |
| 普通用户 | `viewer` | `viewer123` | 查询、导出、敏感字段脱敏 |
| 研究员 | `analyst` | `analyst123` | 行级/时间级/功能权限限制 |

## 2. 登录与退出

1. 打开登录页。
2. 输入用户名和密码。
3. 点击“登录”。
4. 登录成功后进入数据驾驶舱。
5. 点击右上角退出按钮可清除登录态。

异常情况：

- 用户名或密码错误：页面提示登录失败。
- token 过期：系统自动返回登录页。
- 权限不足：页面提示需要管理员权限或隐藏相应入口。

## 3. 数据驾驶舱

驾驶舱用于快速展示系统整体状态：

- 股票标的数量、基金标的数量。
- 股票日线行数、基金净值行数。
- 采集成功率、最近运行记录。
- 行业分布、走势速览。
- WebSocket 实时行情/净值快照。

## 4. 股票行情查询

1. 左侧菜单进入“股票行情”。
2. 选择股票标的，例如 `600519.SH`。
3. 可选择日期范围。
4. 查看 K 线图、收盘线、成交量和行情明细。
5. 管理员可点击采集按钮进行全量或增量采集。
6. 具备导出权限的用户可导出 CSV。

权限说明：普通用户查询时敏感字段 `amount` 会脱敏为 0；管理员可见真实成交额。

## 5. 基金净值查询

1. 左侧菜单进入“基金净值”。
2. 选择基金标的，例如 `510300.OF`。
3. 查看单位净值、累计净值、复权净值和日增长率。
4. 管理员可采集基金净值。
5. 可按权限导出数据。

## 6. 公告舆情

该页面展示公告、新闻、年报、舆情等公开数据源信息。

- 可按代码、类别、情绪标签筛选。
- 管理员可触发公告采集。
- 数据可用于说明题目要求中的“公开数据源抓取”。

截图占位：【图：公告舆情列表，待补】

## 7. 采集任务

管理员可进行任务管理：

1. 点击“新建任务”。
2. 填写任务名称、任务类型、目标代码、采集模式、频率。
3. 选择 `full` 全量或 `incremental` 增量模式。
4. 可使用频率预设：实时、分钟级、日级、周级、季度、手动。
5. 点击“执行”立即运行任务。
6. 点击“日志”查看历史运行记录。

普通用户可以查看运行记录，但不能创建、执行或删除采集任务。

## 8. 数据源接入

数据源页面展示当前系统登记的数据源和命中情况：

- akshare：主要第三方开源数据源。
- tencent / sina：股票行情备用源。
- sample：内置样例兜底数据源。
- announcement：公告/新闻/舆情源。

该页面用于答辩时说明“多渠道数据源接入”和“失败回退机制”。

## 9. 数据质量

数据质量页面展示跨源偏差、异常值、缺失等问题。管理员可对问题进行处理：

- 标记为已修正。
- 标记为忽略。
- 填写处理说明。

该页面对应题目要求中的“跨数据源交叉验证”和“人工数据检查校对”。

## 10. 监控运维

监控页面展示：

- 数据库大小、缓存命中率、采集成功率。
- 近 30 个交易日的数据完整率。
- 当前告警列表。
- API 请求数、平均耗时、错误率。
- 历史告警处理记录。

## 11. 元数据血缘

元数据页面包含两个部分：

1. 数据字典：表名、字段名、字段含义、敏感级别。
2. 数据血缘：每个标的的数据来源、行数、起止日期和最近更新。

该页面用于说明“数据字典、字段敏感级别、血缘追踪、时效性标注”。

## 12. SQL 查询台

仅管理员可访问。系统只允许只读 SELECT 查询，并具备以下限制：

- 禁止 DELETE、UPDATE、INSERT、DROP 等危险语句。
- 禁止多语句。
- 只能查询白名单表。
- 接口强制 LIMIT，防止一次返回过多数据。

## 13. 数据导出

1. 进入“数据导出”页面。
2. 选择数据集：股票日线、基金净值或标的列表。
3. 选择格式：CSV、Excel、Parquet。
4. 可选压缩或加密。
5. 提交后生成导出记录。
6. 点击下载获取文件。

导出记录按用户隔离，普通用户不能下载管理员导出的文件。

## 14. 系统管理与审计

管理员可进入系统管理页面维护：用户、角色、租户。

审计日志页面记录登录、采集、导出、SQL 查询和用户管理等关键操作，便于追踪和答辩展示。

## 15. 推荐答辩演示流程

1. admin 登录，展示驾驶舱。
2. 展示数据源接入，说明多源和样例兜底。
3. 进入采集任务，执行一次股票或公告采集。
4. 进入股票行情，展示 K 线和明细。
5. 进入基金净值，展示净值曲线。
6. 进入数据质量和监控，说明完整性、异常和告警。
7. 导出数据并下载。
8. 展示 SQL 查询台和系统管理。
9. 退出后用 analyst 登录，演示只能看股票、不能导出、不能看基金。
