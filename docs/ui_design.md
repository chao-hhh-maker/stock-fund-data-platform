# 前端 UI 设计文档（ui_design.md）

> 模块 2 交付物 · 页面结构、交互流程、组件设计

## 1. 设计原则

- 信息优先，简洁专业：以表格 + 图表为核心，避免过度装饰。
- 一致性：统一使用 Element Plus 组件与配色，左侧导航 + 顶部栏 + 内容区布局。
- 权限自适应：普通用户隐藏采集 / 导出 / 任务管理等管理员按钮。

## 2. 信息架构

```
登录页 (Login)
└── 主框架 (MainLayout)
    ├── 仪表盘 (Dashboard)   统计卡片 + 最近运行/导出
    ├── 股票查询 (Stocks)    选择器 + 走势图 + 表格 + 采集/导出
    ├── 基金查询 (Funds)     选择器 + 净值图 + 表格 + 采集/导出
    ├── 采集任务 (Tasks)     任务表 + 新建 + 执行 + 日志抽屉
    └── 数据导出 (Exports)   导出记录表 + 新建导出 + 下载
```

## 4. 关键交互

| 交互 | 行为 |
| --- | --- |
| 登录成功 | 保存 token，跳转仪表盘 |
| token 失效 | Axios 拦截器捕获 401，登出并跳登录页 |
| 权限不足 | 拦截器捕获 403，提示「需要管理员账号」 |
| 采集按钮 | 调用 quickCrawl，完成后自动刷新数据与图表 |
| 导出 | 提交导出请求 → 提示成功 → 导出页下载 |
| 涨跌幅着色 | ≥0 红色，<0 绿色（A 股习惯） |

## 5. 组件设计

- `MainLayout.vue`：侧边导航 + 顶部栏 + 内容容器，读取 auth store 渲染角色标签。
- `LineChart.vue`：封装 ECharts，props 为 `categories` + `series`，支持多系列、缩放（dataZoom）、自适应 resize。
- 视图组件（Login/Dashboard/Stocks/Funds/Tasks/Exports）：各自调用 `api` 模块，局部状态用 `ref/reactive`。

## 6. 状态与请求管理

- 全局状态：Pinia `auth` store（token、username、role，持久化到 localStorage）。
- 请求层：`api/http.js`（Axios 实例 + 拦截器）+ `api/index.js`（接口集中定义）。
- 路由守卫：未登录跳转登录页；已登录访问登录页重定向仪表盘。
