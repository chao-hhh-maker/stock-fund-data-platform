## 1. 设计目标

前端采用 Vue 3 + Vite + Element Plus + ECharts 实现，目标不是单纯堆页面，而是为课程答辩提供一条清晰的演示动线：登录 → 数据驾驶舱 → 数据源 → 采集任务 → 股票/基金查询 → 公告舆情 → 数据质量 → 监控告警 → 导出下载 → 系统管理。

设计原则：

1. **信息优先**：金融数据平台以表格、指标卡、走势图、运行日志为主。
2. **权限自适应**：根据登录返回的角色与权限字段控制菜单和按钮。
3. **可演示优先**：每个页面都能在样例数据下稳定展示，不依赖外部网络。
4. **交互闭环**：关键操作后自动刷新数据、运行记录或图表，避免演示时手动找数据。

```text
    ├── 公告舆情 Announcements
    ├── 采集任务 Tasks
    ├── 数据源接入 DataSources
    ├── 数据质量 DataQuality
    ├── 监控运维 Monitor

```

## 3. 页面设计说明

| 页面 | 文件 | 核心内容 | 关键交互 |
| --- | --- | --- | --- |
| 登录页 | `views/Login.vue` | 项目标题、账号密码表单、演示账号快捷填充 | 登录成功后写入 token 并跳转驾驶舱 |
| 主框架 | `layouts/MainLayout.vue` | 侧边导航、顶部用户信息、退出登录 | 权限不同显示不同菜单 |
| 数据驾驶舱 | `views/Dashboard.vue` | 标的数、数据行数、成功率、实时滚动、行业分布、走势速览 | WebSocket / REST 联动展示实时快照 |
| 股票行情 | `views/Stocks.vue` | 标的选择、K 线图、收盘线、行情明细、采集/导出 | 管理员可采集，具备权限者可导出 |
| 基金净值 | `views/Funds.vue` | 单位净值、累计净值、累计涨幅、净值表格 | 查询、采集、导出 |
| 公告舆情 | `views/Announcements.vue` | 公告、新闻、舆情信息流、情绪标签 | 采集公告、按代码/类别筛选 |
| 采集任务 | `views/Tasks.vue` | 任务列表、新建任务、频率预设、运行记录、日志抽屉 | 新建、执行、删除、查看日志 |
| 数据源接入 | `views/DataSources.vue` | akshare、腾讯、新浪、样例源、公告源状态与命中次数 | 展示多源接入和兜底链路 |
| 数据质量 | `views/DataQuality.vue` | 跨源偏差、异常值、缺失问题列表 | 管理员可修正/忽略问题 |
| 监控运维 | `views/Monitor.vue` | 系统指标、完整性检查、当前告警、API 性能、历史告警 | 告警处理和忽略 |
| 元数据血缘 | `views/Metadata.vue` | 数据字典、字段敏感级别、来源、行数、时间范围 | 展示字段级权限依据 |
| SQL 查询台 | `views/SqlQuery.vue` | 只读 SQL 输入、预设 SQL、结果表 | 管理员执行受控查询 |
| 数据导出 | `views/Exports.vue` | 导出记录、格式、压缩/加密、下载 | 导出、下载、记录追踪 |
| 系统管理 | `views/Admin.vue` | 用户、角色、租户管理 | 管理员 CRUD |
| 审计日志 | `views/Audit.vue` | 登录、采集、导出、SQL、用户管理日志 | 管理员查询审计 |

## 5. 状态与请求管理

- `stores/auth.js`：集中保存 token、username、role、data_scope、can_export、can_view_sensitive 等权限字段。
- `api/http.js`：Axios 实例，统一处理 token 注入、401 退出登录、403 权限提示和错误消息解析。
- `api/index.js`：所有后端 API 的前端调用封装，避免页面内散落 URL。
- `router/index.js`：路由守卫控制未登录跳转、登录后重定向、管理员页面访问控制。
